Predicting Boston Property Price

Wednesday, Jul 1, 2020| Tags: Trulia, data scrapping, model productization

Introduction

Predict Boston property price by analyzing data scraped from trulia.com using advanced regression models. I will go in-depth for all the processes of this project: from using Beautiful Soup to scrape data, to analyze and compare different regression models, and building an Application Programming Interface (API) using a micro web framework in Python: Flask.

For in-depth step-by-step code analysis, the repository for this project can be found here.

My goal for this project is to use the best regression model to predict Boston housing prices on Trulia based on the number of features that the source provides. By the end of the project, I am hoping to be able to answer these questions:

  1. What are the features that have a high effect on the rent of a property? Does the neighborhood of the property have any effect?
  2. How good is our prediction for the rent of the property?

Since I have done a simple regression model with my previous project (“GDP_Fertility_Mortality_Relation”), I want to improve my skill sets and prediction accuracy even more. In this project, I have three main focuses:

  1. Learn how to scrape a website with Python.
  2. Understand the importance of Exploratory Data Analysis (EDA) and how it contributes to the outcome.
  3. Introduce the productization stage into my project.

One more improvement I apply in this project is using different regression models and improve its performance by fine fine-tuning the parameters using GridSearchCV.

General Project Outline

  1. Data collection: use BeautifulSoup to scrape property data from Trulia, a popular real estate website. Gather all listed variables that can be used for data analysis.
  2. Data cleaning: read in data and prepare it for data analysis; steps include: tidy up the categorical features, deal with null values, etc.
  3. Exploratory data analysis (EDA): examine the cleaned data and its trends so we can choose an appropriate model that can be applied.
  4. Model building: determine which model (Linear, Lasso, Random Forest, XGBoost) work best (that return the smallest error) and tune the model with different parameters using GridSearchCV.
  5. Productionize model: create a local API to get quick access to the regression model with a given input set.

Disclaimer: this data set is used for educational puspose.

To evaluate the performance of our model, I will use the mean absolute error (MAE) as the metric for this project. The reason for choosing this metric is because it can represent, on average, how far off our prediction is.

Data Collection

Starting with data collection, I have never learned how to scrape a website before. Therefore, I want to give the acknowledgment of the code used for scrapping to Karishma Parashar (the GitHub repository for her code can be found here). Her code gives a nice outline of the process. Though there are some bugs that I needed to fix for the code to run properly and to get the type of data and the amount I needed. I started the scrapping procedure on June 25h, 2020. My goal is to scrape, at most, 240 records (not all search terms will result in at least 240 records) from each neighborhood of Boston and convert it to a CSV file. I have collected 3,894 records and 12 different variables. One thing to note is the “feature” variable: I could not code the scrapping process to attain all features because of the “See All” button that hides some of the data. Do not rely heavily on this variable when doing analysis.

Neighborhoods as search key terms: East Boston, Charlestown, Allston, North End, West End, Downtown, Chinatown, Back Bay/ Beacon Hill, South Boston, South End, Fenway, Mission Hill, Roxbury, Dorchester, Jamaica Plain, Mattapan, Roslindale, West Roxbury, Hyde Park.

Here is a list of variables that I scrapped from Trulia:

  1. rent: the rent of the property for 1 month.
  2. address: the address of the property.
  3. area: neighborhood the property located.
  4. bed: number of beds provided.
  5. bath: number of bathrooms provided.
  6. school: number of school around the area.
  7. crime: crime rate of the area.
  8. commute: percentage of people commute by car.
  9. shop_eat: number of shops and restaurants in the area.
  10. description: description of the property.
  11. feature: items that property provides (heating, laundry, etc.).
  12. URL: link to the property.

EDA

After preparing the data, next is the data interpretation in the EDA stage. First, I want to look at the general information of the variables. From figure 2, there are three pieces of information I can see: I have about 3259 records of property, there is no missing value, and the data set only consists of 3 categorical variables and 20 numerical variables.

Next, I will perform univariate analysis on the target variable: “rent” by caculating the skewness and kurtosis of the variableand, and plotting the value and examine if the distribution shape is normal. From the table and graph below, we can see that: since the original data has high positve skewness and kurtosis (the curve is formed by a huge cluster of mid-range properties and few expensive properties that cause it to have a right skew), I normalize the data by performing log transformation and it resulted very close to a normal distribution.

I also perform multivariate analysis on the target variable by having “bed” and “bath” as an additional dependent variable and see the relationship between rent and the number of beds and bathrooms a property has. With the plot below, there is an upward trend between the relationships.

To decrease the skewness and make the general distribution even more normal, I identify the outliers using Z-score and remove it from the data set. Here is the “rent” distribution resulting from having the outliers removed.

Heatmap is a very useful tool to see how variables are correlated with each other. From figure 9, there are many interesting details that we need to pay attention to:

  1. “bed”, “bath”, “restaurant”, “grocery”, and “nightlife”, accordingly, have the highest correlation with the target variable. Though they are less correlated than expected.

  2. All the school variables are highly correlated with each other. This is predicted since the values were derived from a single variable during our cleaning phase. Similar can be said with “restaurant”, “grocery”, and “nightlife” (they came from a variable that is called “shop_eat”)

  3. All the features variables interesting have a lower correlation with the target variable. Keep in mind that we will not use much of these data.

Observed only those numerical variables that have a high correlation with the target variable with a zoomed heatmap. Since we have mentioned that ‘restaurant’, ‘grocery’, and ‘nightlife’ are pretty similar, we can remove two of the variables and keep ‘grocery’ for now for analysis (figure 12)

Next, I examine the categorical variable. I explore the “area” variable with the assumption that: depends on the neighborhood, the general trend for the rent of a property might be different. Here, I make a pie chart shows the proportion of area that each neighborhood takes up and a boxplot of the relationship between “area” and “rent”. By looking at the mean and its ranges, there seems to be some correlation. Another thing worth noting is the number of outliers presented in almost every neighborhood.

My final task for this stage is to examine “property_type” and “crime” and their relation with “rent”. Because there is an imbalance proportion of data for “property_type” (apartment 3%, condo > 0.1%, multi 96%, single 1%, townhouse and condo are > 1%), it can be hard to examine the relationship accurately. Based on the graph, we can see a representation of what it would look like: multi-family and townhouse have about the same median, a little bit higher than that are single-family; although apartment seems to have the same median as a condo, I believe if we have enough data points, condos would be the property with the highest rent.

As for “crime” in figure 17, Moderate and Low crime rates have the same median and range. Although the High rate does have a similar median but interestingly has a higher range. The two most surprising ones are the Lowest and Highest rates: while Lowest one also has the lowest median and, arguably, range, the Highest rate has the highest median. I expected it to be the opposite of these two. Based on this, maybe more expensive properties attract more crime than lower ones?

Regression Model

Split the data to training (80%) and testing (20%) sets in a stratify fashion with strata are the different neighborhoods. The reason for this is because I want to have coverage of the entire area in the Boston city and some neighborhood have significantly lower data than others.

Testing assumptions:

  1. Normality/ Linearity: when I was investigating our rent variable during the EDA stage, I have transformed the data into log so it can prevent underfitting. But I tested it again for our train data to be sure also (figure 18 and 19).
  2. Homoscedasticity: I included all variables to eliminate heteroscedasticity.
  3. Absence of Multicollinearity: though I have tried to remove variables that are highly correlated to each other, I found in the result of regression models that by including every variable I was able to get a higher accuracy rate.

Ordinary least squares (OLS) regression, by being one of the simplest estimators for a simple model, is easier to implement and interpret compare to other sophisticated ones. Although it might not work well with my project that has many variables and correlations to the target is weak, I gave it a shot. Besides the table of correlation that I already have a general picture of what it is like, a statistical measurement that I am interested in is the “R-squared” (coefficient of determination) of 0.661. This means that our model fits well and we can predict about 66.1% of our trained data set. Keep in mind that although the difference between “R-squared” and “Adj. R-squared” (0.657)is not too significant, but it let us know that there are some irrelevant features that we have included in our model. Another interesting measurement is “F-statistics” that has value above 100 (152.3) and “Prop (F-statistics)” that has a value below 0.05 (0.00). By having these two measurements meeting the condition, it means that there is a good linear relationship between the target and all the feature variables.

Throughout all of the model, I will implement cross validation to eliminate the probability that the model might be over fitting or contains bias

Multiple linear regression (this is the same model of what we have discussed above, but I use mean absolute error here as my measurement of the model accuracy) returns a MAE of -393.7313 for trained data set

Lasso (least absolute shrinkage and selection operator) regression contrasts with the model above. I tested out the Lasso regression model because of its ability to analyze data set with large features very well. That is not the only reason for applying in this model because I want to see how its L1 regularization technique works (helps with eliminating overfitting). For this model, I have tested out a range of alpha from 1 to 100 with an increment of 10. By finding the maximum error of the curve of the plot below, we have alpha = 1.7. It returned a MAE of -392.8202 for trained data set.

Random forest regression applies some different techniques to predict the data: it uses multiple decision tree and bagging (or bootstrap aggregation) to better understand the bias and variance of the data. Because it also works well with a large data set, I applied this model to our data to see how it does compare to other models. It returned a MAE of -306.6928 for trained data set.

XGBoost is known for its ability to outperform any of its competitors, XGBoost is my last algorithm of choice to predict our data. Different from random forest regression, it uses boosting technique (combining weak learners to strong ones to improve prediction accuracy). It is also powerful for a wide range of parameters to really fine-tune the algorithm. It returned a MAE of -343.4491 for trained data set.

Overall Model Performance

Before finalizing our model, I made three different adjustments to the data to see if it can improve the prediction:

  1. Size: initially, after finished cleaning the data, I trimmed the data set from 3000+ records to about 759 records. I trimmed it based on the proportion of the area that each neighborhood takes relative to the area of the city. The reason why the size is small is because of neighborhoods, such as Hyde Park and Mattapan, which have very few records on Trulia. But it turned out that it does not give a very good prediction as if I leave the size as it is. This concluded that the larger the sample size is the better the prediction is.
  2. Stratification: in order for our sample to accurately represent the city of Boston, I make sure all neighborhoods are present in the training data set. As it turned out, this also resulted in a better prediction for our model.
  3. Eliminate irrelevant features: recalling back to our heat maps from above, there are only a few variables that give me the correlation with the target variable that is above 0.2. I thought, by removing the irrelevant features and those that highly correlated to each other, would increase the performance. But it did not turn about to be like that. I am still unsure of why this is the case. But by including all variables, it seems to perform better (the larger the number of features is the better?)

There are two things in the list of our prediction models’ outcomes that surprised me: there is only 0.9 improvement in lasso regression compare to the multiple linear regression and the random forest regression outperformed the XGBoost algorithm by 36.7 dollar. Because of this, I will apply an exhaustive GridSearchCV to search for the best parameters for the random forest regression. Although I have tried to exhausted tuning with many more parameters such as “max_depth” and “min_samples_leaf”, it took too much time so I did a simple one instead (so the difference of the result is not huge). With the best parameters for the best performance model through the training data set, now it is the time to implement it to our test data set too see how well our model can predict the prices of property in Boston:

  1. Multiple linear regression: 1469.1753603498644
  2. Lasso regression: 393.3840532780861
  3. XGBoost regression: 403.93248876337
  4. Random forest regression regression (using best parameters through GridSearchCV): 316.51083201892743

Conclusion

With our best prediction model returns the MAE of 316.51 for the test data set, the model does predict the property’s price according to the features that are used quite accurately.
